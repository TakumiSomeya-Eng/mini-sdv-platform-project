# M8 FRD — マルチビークル・フリート管理

## L3 Domain Hypothesis (ドメインルール)

| ID | ルール | 根拠 |
|----|--------|------|
| DR-80 | `fleet-simulator` は Kuksa Databroker への書き込み禁止（READ ONLY） | 信号源の権威はECU Simulatorのみ |
| DR-81 | vehicle-002/003 は MQTT + InfluxDB に直接書き込む（CAN/Databroker バイパス） | クラウド側フリート管理の現実的な模倣。各車両は独立してクラウドエンドポイントに送信する |
| DR-82 | 各車両のシグナル値は互いに独立したランダム値を生成する | 実フリートは車両ごとに異なる走行状態を持つ |
| DR-83 | MQTT トピック形式: `sdv/{vehicle_id}/telemetry`（vehicle-001 と同形式） | 既存の Mosquitto サブスクリプション構造と整合 |
| DR-84 | InfluxDB の measurement は `vehicle_signals`（既存スキーマを維持） | M7 で確立したスキーマを拡張するだけでフリート対応できることを示す |
| DR-85 | OTA 一括配信は `scripts/fleet-ota-trigger.sh` で実装 | 単一スクリプトで全車両に同時配信する運用体験 |
| DR-86 | Grafana Fleet ダッシュボードは `fleet_overview.json` として既存ダッシュボードと分離 | 単車ダッシュボードとフリートダッシュボードの責務を分ける |
| DR-87 | fleet-simulator は vehicle-002/003 を**同一プロセス内**でスレッド並行実行する | コンテナ数を最小化し、docker-compose の複雑さを抑える |

## L4 Interaction Hypothesis (UI/UX)

### 4.1 Grafana Fleet Overview ダッシュボード構成

```
┌─────────────────────────────────────────────────────────────────────┐
│  Fleet Overview — Last 30 min  [auto-refresh: 5s]                   │
├─────────────────────────────────────────────────────────────────────┤
│  Vehicle Speed (km/h) — 全3台オーバーレイ                            │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  vehicle-001 ───  vehicle-002 ───  vehicle-003 ───          │    │
│  └─────────────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────────────┤
│  Battery SoC (%) — 全3台オーバーレイ                                 │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  vehicle-001 ───  vehicle-002 ───  vehicle-003 ───          │    │
│  └─────────────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────────────┤
│  Cabin Temperature (°C) — 全3台オーバーレイ                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  vehicle-001 ───  vehicle-002 ───  vehicle-003 ───          │    │
│  └─────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

- 1 パネル = 1 シグナル、全車両の折れ線を重ねて表示
- 凡例に vehicle_id タグ値を自動表示（Flux の `group()` で分離）
- URL: http://localhost:3000（既存 Grafana）

### 4.2 OTA 一括配信フロー

```bash
# scripts/fleet-ota-trigger.sh
# 1コマンドで全車両に OTA 配信

./scripts/fleet-ota-trigger.sh 1.1.0
# → curl -X POST http://localhost:8080/release/1.1.0
# → MQTT トピック sdv/+/ota/status で3台分の進捗を確認
```
